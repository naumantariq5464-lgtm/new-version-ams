About AMS

Built to Digitize Attendance
for the Modern Classroom

The Attendance Management System (AMS) replaces physical registers with a secure, structured, and scalable digital solution — designed with strict data integrity and role-based control at its core.

"Every student deserves transparent access to their own data. Every teacher deserves full ownership of their classroom."

AMS was designed with one guiding principle: data should belong to the people who generate it. Teachers own their students, subjects, and attendance records. Students own their personal attendance history. No cross-contamination, no unauthorized access — ever.

Our Mission

Why We Built AMS

Manual attendance registers are error-prone, time-consuming, and impossible to audit. AMS solves all of that.

Digitize Every Record

Transform paper registers into structured, queryable database records with complete audit history.

Semester-Scoped Isolation

Every record is bound to a specific semester, preventing cross-semester data leakage.

Student Transparency

Students can instantly view their attendance percentage per subject and download PDF reports.

Design Principles

Core Principles Guiding AMS

01
Semester Isolation

Every data record is bound to a semester. No cross-semester access is permitted at any layer — not at the API, not at the database.

Critical
02
Teacher Ownership

Each teacher owns and manages only their students, subjects, and attendance records. Strict teacher ID filtering at ORM level guarantees this.

Critical
03
Student Read-Only

Students can view and download their attendance but cannot modify any record. All write operations on attendance are blocked for the student role.

Enforced
04
Security First

All endpoints are secured with JWT Bearer tokens. Passwords are hashed using Bcrypt with salt rounds ≥ 12. HTTPS is enforced in production.

Critical
05
Scalability

Modular architecture using FastAPI and PostgreSQL (Neon Cloud) allows the system to scale to hundreds of teachers and thousands of students.

Built-in
06
Clean UX

Bootstrap 5 responsive UI with intuitive flows — usable without training. Mobile-first design ensures accessibility on any device.

Security

Security Built Into the Core

From JWT tokens to database-level filtering, every layer of AMS is designed to protect your data and enforce strict access boundaries.

JWT Authentication

Stateless Bearer tokens with 24h expiry. All protected routes validate the token and extract the user's role before processing.

Bcrypt Password Hashing

Passwords are never stored in plain text. Bcrypt with salt rounds ≥ 12 ensures brute-force resistance for every account.

Role-Based Access Control

Middleware validates each user's role from their JWT before processing any request. Teachers and students have entirely separate API capabilities.

ORM-Level Data Isolation

All database queries include teacher_id = current_user.id and semester = current_user.semester filters automatically — no raw SQL, no injection risk.

Pydantic Input Validation

All incoming API request bodies are validated by Pydantic models before any processing occurs — type-safe from edge to database.
